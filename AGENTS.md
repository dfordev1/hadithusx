# Project continuity instructions

Before making product changes, read:

1. `docs/GOAL.md`
2. `docs/DONE.md`
3. `docs/NEXT.md`

Before every interface change, also read `docs/THEME.md`. The white scholarly theme is a permanent product requirement.

After verified work:

- update `docs/DONE.md` with completed outcomes;
- remove or revise the corresponding work in `docs/NEXT.md`;
- keep claims honest about the difference between software readiness and scholarly verification;
- run the relevant checks before describing work as complete.
